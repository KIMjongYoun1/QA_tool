# QA 통합 관리 툴

테스트 케이스 관리부터 결과서 출력, 배포 히스토리 집계까지 별도 서버·DB 없이 동작하는 로컬 QA 툴입니다.

---

## 설계 철학

> **"서버 없이, DB 없이, 파일 두 개로 QA 업무 전체를 처리한다."**

이 툴은 의도적으로 HTML과 Python을 합치지 않았습니다.

- **서버 불필요** — Flask, Node 같은 백엔드 서버를 띄우지 않습니다. HTML 파일을 더블클릭하는 것만으로 즉시 실행됩니다.
- **DB 불필요** — MySQL, SQLite 같은 데이터베이스를 사용하지 않습니다. 데이터는 브라우저 localStorage와 엑셀 파일로만 관리합니다.
- **경량화** — 설치 과정이 없습니다. 파일을 폴더에 넣는 것만으로 어디서든 동일하게 동작합니다.
- **역할 분리** — HTML은 QA 수행과 결과서 출력에 집중하고, Python은 히스토리 집계에 집중합니다. 각자 잘하는 것만 합니다.
- **이식성** — USB에 넣어 다른 PC에서 열어도, 팀원에게 파일로 전달해도 그대로 동작합니다.

---

## 폴더 구조

압축 해제 후 아래와 같은 구조로 구성되어 있습니다.

```
QA/
├── README.md                   ← 이 파일
├── qa_report.html              ← 메인 QA 툴
├── exceljs.min.js              ← 오프라인 엑셀 생성 라이브러리 (qa_report.html과 같은 폴더 필수)
├── qa_history.py               ← 배포 히스토리 집계 Python 툴
├── qa_history_결과.xlsx         ← Python 툴 실행 시 자동 생성되는 파일
│
├── qa_exports/                 ← HTML 툴에서 내려받은 엑셀을 여기에 저장
│   ├── QA결과_20250210_v1_0_0.xlsx
│   ├── QA결과_20250318_v1_1_0.xlsx
│   └── ...
│
└── 샘플/                       ← 결과물 샘플 (참고용)
    ├── 테스트결과서_샘플_v2.4.0.xlsx
    ├── 테스트결과서_샘플_v2.4.0.pdf
    ├── 배포결과서_샘플_v2.4.0.xlsx
    └── 배포결과서_샘플_v2.4.0.pdf
```

> **주의**: `qa_report.html` 과 `exceljs.min.js` 는 반드시 **같은 폴더**에 있어야 합니다.
> 파일을 분리하거나 html 파일만 다른 곳에 옮기면 엑셀 다운로드가 동작하지 않습니다.

---

## 사용 흐름

```
[1단계] QA 수행
qa_report.html 더블클릭 → 케이스 입력 → 테스트 수행 → 엑셀 다운로드

[2단계] 엑셀 누적
다운받은 엑셀을 QA/qa_exports/ 폴더에 저장

[3단계] 히스토리 집계
터미널에서 qa_history.py 실행 → qa_history_결과.xlsx 생성
```

---

## 1단계 — qa_report.html 사용법

### 실행 방법

`qa_report.html` 파일을 **더블클릭**하면 브라우저에서 바로 열립니다.

- **Chrome 사용 권장** — 데이터가 브라우저 localStorage에 저장되므로 브라우저를 고정해서 사용하세요.
- 인터넷 연결 없이도 동작합니다 (`exceljs.min.js` 가 같은 폴더에 있어야 함).

### 탭 구성

**📝 케이스 관리**
- 테스트 케이스 추가 / 수정 / 삭제
- Pass / Fail / N/A 결과 입력
- 드래그 앤 드롭으로 순서 변경

**📋 테스트결과서**
- 표지 정보 입력 (프로젝트명, 버전, 작성자 등)
- 카테고리 분류 후 케이스 배정
- 인쇄(PDF) / 엑셀 다운로드

**🚀 배포결과서**
- 배포 정보 입력 (버전, 환경, 배포자, 배포 시작/완료 시각 등)
- 배포 항목별 완료 / 미완료 처리
- **일괄 완료 처리** 버튼으로 전체 항목 한 번에 완료 처리
- 인쇄(PDF) / 엑셀 다운로드

### 엑셀 다운로드 위치

브라우저 기본 다운로드 폴더에 저장됩니다 (보통 `~/Downloads/`).
다운로드된 파일을 `QA/qa_exports/` 폴더로 이동시켜 주세요.

### 데이터 백업 / 복원

- **💾 백업** : 현재 데이터를 JSON 파일로 저장 (다운로드 폴더에 생성)
- **📂 복원** : JSON 파일 선택 시 데이터 자동 복구

> PC를 바꾸거나 브라우저를 변경할 때는 반드시 **백업 → 복원** 순서로 데이터를 이전하세요.

## 2단계 — 엑셀 파일 누적

HTML 툴에서 엑셀을 다운받을 때마다 아래 경로에 저장합니다.

```
QA/
└── qa_exports/
    ├── QA결과_20260101_v1_0_0.xlsx   ← 1월 배포
    ├── QA결과_20260215_v1_1_0.xlsx   ← 2월 배포
    └── QA결과_20260310_v2_0_0.xlsx   ← 3월 배포
```

파일명 규칙은 자유롭게 사용해도 되며, `.xlsx` 파일이면 모두 자동으로 읽힙니다.

---

## 3단계 — qa_history.py 사용법

### 사전 준비 (최초 1회)

Python 3가 설치되어 있어야 합니다.
터미널을 열고 아래 명령어로 필요한 라이브러리를 설치합니다.

```bash
pip3 install openpyxl
```

### 실행 방법

터미널에서 QA 폴더로 이동한 뒤 실행합니다.

**Mac / Linux**
```bash
cd /경로/QA
python3 qa_history.py
```

**Windows**
```bash
cd C:\경로\QA
python qa_history.py
```

### 실행 예시

```
==================================================
  QA 히스토리 통합 툴
==================================================
[+] 10개 파일 발견
    ✓ QA결과_20250210_v1_0_0.xlsx
    ✓ QA결과_20250318_v1_1_0.xlsx
    ...

[+] 출력 파일 생성 중...
[✓] 완료: ./qa_history_결과.xlsx

출력 시트:
  - 배포_히스토리 : 배포별 요약 (10건)
  - 케이스_마스터 : 누적 케이스 통계 (15개)
  - 케이스_이력   : 케이스 × 배포 교차 이력 (69건)
  - 배포_상세     : 배포 수행 결과 원본 (69건)
```

실행이 완료되면 `QA/qa_history_결과.xlsx` 파일이 생성됩니다.

### 출력 파일: `qa_history_결과.xlsx`

| 시트 | 내용 |
|---|---|
| 배포_히스토리 | 배포건별 요약 — 버전 / 날짜 / 환경 / 완료율 |
| 케이스_마스터 | 케이스별 누적 통계 — 총 실행 / PASS / FAIL / 통과율 / 배포별 이력 |
| 케이스_이력 | 케이스 × 배포 교차 이력 — 언제 어떤 배포에서 Pass/Fail |
| 배포_상세 | 전체 배포 수행 결과 원본 |

엑셀이 쌓일수록 케이스_마스터의 배포별 이력 컬럼이 자동으로 누적됩니다.

---

## 주의사항

- `qa_report.html` 과 `exceljs.min.js` 는 **같은 폴더**에 있어야 합니다
- 데이터는 브라우저 localStorage에 저장되므로 **시크릿 모드에서는 데이터가 유지되지 않습니다**
- `qa_exports/` 폴더 안에 열려 있는 엑셀 파일(`~$` 로 시작하는 임시파일)이 있으면 자동으로 건너뜁니다
- `qa_history_결과.xlsx` 는 실행할 때마다 **덮어써집니다** — 보관이 필요하면 별도 저장하세요
